# FinSight-AI Master Data Flow & Presentation Script

This document provides a bullet-proof, granular map of **every single sidebar link** across the Accountant, Auditor, Manager, and Compliance roles. It tells you exactly what data each page displays, which database table it pulls from, and **exactly how to add or edit that data** to demonstrate the platform.

---

## 1. Accountant Role

| Sidebar Menu Item | What It Displays | Data Source (Backend) | How to Add / Edit Data |
| :--- | :--- | :--- | :--- |
| **Dashboard** | High-level metrics: Pending Tasks, Journals, Bank Reconciliations, and AI Risk Alerts. | `Task`, `JournalEntry`, `Reconciliation` tables. | Log into Auditor or Manager to create tasks or run reconciliations. |
| **Upload Documents** | OCR extraction tool for parsing invoices. | Integrates with AWS/OpenAI OCR engine. | **ADD:** Upload a PDF invoice. It automatically extracts data. Click **"Save to Ledger"** to create a Draft Journal Entry, or **"Save to My Reports"** to generate a Narrative Report. |
| **Tasks -> Accruals** | Displays tasks containing the word "accrual" assigned to the Accountant. | `Task` table (`GET /tasks/accountant`). | **ADD:** Log in as **Manager** -> Tasks -> Click "Delegate Task". Set title to "Accrual Adjustment" and assign it to the Accountant. |
| **Tasks -> Tax** | Displays tasks containing "tax" or "gst" assigned to the Accountant. | `Task` table (`GET /tasks/accountant`). | **ADD:** Log in as **Manager** -> Tasks -> Click "Delegate Task". Set title to "Tax Adjustment". |
| **Reconciliation -> Bank** | Displays Bank/Ledger reconciliation records. | `Reconciliation` table (`GET /reconcile`). | **ADD:** Use the **"Run AI Reconciliation"** form right on this page. Enter a Bank balance and Ledger balance. The AI will flag it as MATCHED or DISCREPANCY. |
| **Reconciliation -> Vendor** | Displays Vendor/Ledger reconciliation records. | `Reconciliation` table (`GET /reconcile`). | **ADD:** Use the **"Run AI Reconciliation"** form right on this page to log vendor discrepancies. |
| **Journal Entries** | Displays all journal entries (draft, posted, flagged). | `JournalEntry` table (`GET /journals`). | **ADD:** Click the **"Add New Journal Entry"** button on this page. Or use the "Upload Documents" OCR feature to automatically create one. |
| **Alerts** | Displays High/Medium/Low risk alerts generated by AI. | `JournalEntry` table (where `riskScore > 50`). | **ADD:** Log in as **Auditor** -> Journals Review -> Click **"Analyze"** on a Draft journal. If the AI flags it, it will appear here as an Alert. |
| **My Reports** | Displays AI-generated narrative reports and summaries. | `Report` table (`GET /narratives/reports`). | **ADD:** Use the **"Upload Documents"** page -> Process a PDF -> Click **"Save to My Reports"**. |
| **AI Copilot** | Chat interface for general ledger queries. | LangChain / OpenAI API. | Direct interaction. |

## 2. Auditor Role

| Sidebar Menu Item | What It Displays | Data Source (Backend) | How to Add / Edit Data |
| :--- | :--- | :--- | :--- |
| **Dashboard** | Audit overview, pending clearances, and flagged journals. | `Task`, `JournalEntry` tables. | Driven by Accountant's journals and Manager's tasks. |
| **Audit Trail** | System-wide immutable logs of who did what. | `AuditLog` table. | **ADD:** This is automated. For example, when Compliance uploads Evidence, a log is automatically generated here. |
| **Clearance** | Displays audit clearance tasks. | `Task` table. | **ADD:** Click **"Assign Task"** from the Auditor Tasks page and specify it as a Clearance task. |
| **Compliance Check** | Summary of control statuses and policies. | `RiskControl`, `Policy` tables. | **ADD:** Log in as **Compliance** -> Internal Controls -> "Add Control". |
| **Department Audit Overview** | Aggregates logs and issues by department. | `AuditLog`, `JournalEntry` tables. | Automated aggregation of system activity. |
| **Flags & Red Alerts** | Displays critical system alerts and anomalies. | `JournalEntry` (Flagged). | **ADD:** Generated when the Auditor runs the AI "Analyze" tool on a journal and it finds fraud. |
| **Journals -> Details** | Deep dive view of all ledgers. | `JournalEntry` table. | **ADD:** Log in as **Accountant** -> Journal Entries -> Add. |
| **Journals -> Exceptions** | View journals explicitly marked as "FLAGGED". | `JournalEntry` table. | **ADD:** Click "Analyze" on a journal. If AI flags it, it appears here. |
| **Journals -> Review** | Pending Draft journals requiring Auditor action. | `JournalEntry` table. | **EDIT:** Click **"Analyze"** to let the AI score the journal, or manually "Approve"/"Reject". |
| **Reconciliations -> Issues** | Displays Reconciliations marked as DISCREPANCY. | `Reconciliation` table. | **ADD:** Log in as **Accountant** -> Bank Reconciliation -> Run AI on mismatched balances. |
| **Tasks -> Task Audit View** | Displays tasks created by the Auditor. | `Task` table. | **ADD:** Click the **"Assign Task"** button on this page to delegate work to Managers or Accountants. |

## 3. Manager Role

| Sidebar Menu Item | What It Displays | Data Source (Backend) | How to Add / Edit Data |
| :--- | :--- | :--- | :--- |
| **Dashboard** | Manager KPI overview (Revenue, Expenses, Pending Tasks). | `Task`, `Report` tables. | Aggregated data. |
| **Audit Trail** | View of the system audit logs. | `AuditLog` table. | Automated system logging. |
| **Compliance Overview** | High-level view of compliance policies. | `Policy` table. | **ADD:** Log in as **Compliance** -> Policies -> "Add Policy". |
| **Departments -> Overview** | List of system departments based on user roles. | `User` table (grouped by role). | Automated based on active registered users in the database. |
| **Fiscal Close** | Displays "Period Close" tasks and statuses. | `Task` table. | **ADD:** Create tasks labeled "Fiscal Close" using the Manager Task page. |
| **Forecasting -> Forecast Scenarios** | AI predictive revenue/expense modeling. | LangChain Engine (`GET /engine/forecast`). | **ADD:** Click the **"Re-Analyze Live Data"** button to trigger the Python AI engine to generate new scenarios. |
| **Forecasting -> What-If** | AI strategic modeling for hypothetical scenarios. | LangChain Engine (`GET /engine/what-if`). | **ADD:** Click the **"Run AI Scenarios"** button to instruct the AI agent to model new data. |
| **Narratives** | Displays narrative reports generated by the team. | `Report` table. | **ADD:** Log in as **Accountant** -> Upload Documents -> process invoice -> Save to Reports. |
| **Tasks** | Displays tasks assigned to the Manager. | `Task` table. | **ADD:** Click the **"Delegate Task"** button on this page to create tasks for yourself or others. Click **"Resolve"** to finish them. |

## 4. Compliance Role

| Sidebar Menu Item | What It Displays | Data Source (Backend) | How to Add / Edit Data |
| :--- | :--- | :--- | :--- |
| **Dashboard** | Compliance metrics, untested controls, policy violations. | `RiskControl`, `Policy` tables. | Automated aggregation. |
| **Dept Clearance Panel** | Displays Tasks pending department clearance approval. | `Task` table. | **ADD:** Log in as Auditor or Manager, Assign Task to the "Compliance" user. **EDIT:** Click **"Approve & Clear"** on this page to resolve it. |
| **Monitoring -> Overview** | AI assessment of departmental risk heuristics. | `JournalEntry` flags by Dept. | **ADD:** Log in as Auditor -> Analyze Journals -> Generate Flags. |
| **Monitoring -> Red Flags** | Critical flagged journal entries. | `JournalEntry` table. | **ADD:** Log in as Auditor -> Analyze Journals -> Generate Flags. |
| **Controls -> Control Evidence** | Registry of evidence documents attached to controls. | `Document`, `RiskControl`. | **ADD:** Click the **"Upload"** button on this page to attach a PDF document to an existing control. |
| **Controls -> Control Testing** | Registry of past control test iterations. | `RiskControl` table. | **ADD:** Click the **"Run Test"** button to simulate an automated compliance check. |
| **Controls -> Internal Controls** | The master registry of all organizational controls. | `RiskControl` table. | **ADD:** Click the **"Add Control"** button to create a new governance control. |
| **Policies -> All Policies** | Master registry of corporate policies. | `Policy` table. | **ADD:** Click the **"Add Policy"** button to draft a new corporate policy into the database. |
| **Policies -> Policy Details** | Detailed view of a specific policy. | `Policy` table. | Read-only view of added policies. |
| **Policies -> Violations** | Displays tasks or alerts related to policy breaches. | `Task` table. | Derived from compliance task assignments. |
| **Regulatory -> Reports** | Downloadable compliance PDF reports. | PDF Generator (`GET /reports/:id/download`). | **EDIT:** Click on a report to automatically compile a live PDF from the database. |
| **Regulatory -> Requirements** | External regulatory standards view. | System constants. | Read-only reference table. |
